# 题目二：AI 语音绘图工具

---

## 一、项目要求

请开发一款纯语音控制的绘图工具。

**要求：**
- 用户不能使用鼠标或键盘，仅通过语音指令完成绘图创作
- 综合考虑指令理解的准确性与容错性
- 优化语音到绘图操作的响应延迟
- 具备复杂指令的拆解与执行能力

**额外交付物：** 实现应用的同时，提交一份设计文档，记录：
1. 计划支持哪些指令能力
2. 最终实现了哪些
3. 未完成部分的原因说明

> ⚠️ 请选择本批次提交作品的同学们注意，确保自己的议题方向符合本批次议题要求。若提交之前批次议题的作品，将视为无效作品。
>
> 这是本次招募的最后一批议题，请感兴趣的同学抓紧报名，不要错过了哦。

## 三、作品提交规则

### 3.1 须提交内容

- 公开的 GitHub / Gitee 仓库（过程保密防抄袭；提交截止后可公开访问，以便评委评审作品）
- Demo 视频
- README 文档

### 3.2 作品有效性

- 作品需符合所选议题方向，并确保自主完成无抄袭
- **全周期持续交付**，严禁临尾"突击提交"
  - 从议题发布之日起，开发周期内保持持续的 PR 记录和 commit 提交
  - 仅在最后一天一次性导入所有代码的作品，将直接视为无效作品
- 所有 commit 时间戳必须落在所选批次的开始与截止时间之内，否则视为无效作品

**其他无效情形：**

1. PR 描述空白或与实际代码变更严重不符
2. 引用了第三方库或框架，没在 README 中列明依赖，且未说明原创功能部分
3. 复用了自己过去的代码片段，没在 PR 描述中注明来源

### 3.3 PR（Pull Request）提交规范

请基于 PR 添加新功能。

**基本原则：**

- 每个 PR **只做一件事**：只实现或修改单一功能
- 鼓励尽可能小、粒度尽可能细的 PR
- 大功能应拆分为多个独立 PR 分步提交

**PR 标题与描述需清晰完整，包含：**

| 字段 | 说明 |
|------|------|
| ① 标题 | 一句话说明本 PR 新增/修改了什么 |
| ② 功能描述 | 说明该功能的作用与使用方式 |
| ③ 实现思路 | 简要说明技术选型或核心实现逻辑 |
| ④ 测试方式 | 如何验证该功能正常运行 |

**重要：** PR 合并后，主分支代码需保持可运行状态，评委在任意时间查看应能复现演示效果。

### 3.4 Demo 视频要求

- 请用声音完整讲解、展示作品主要功能和效果，覆盖核心模块
- 上传至可访问的外部平台（如 Bilibili / 云盘等）
- 链接放到代码仓库的 README 中，确保视频可播放

### 3.5 其他要求

- **多人组队**：每队仅需提交一个代码仓库地址。队员均须使用各自账号提交 commit，PR 清晰描述各自分工
- **多模块项目**：如项目包含多个独立模块，应提交至同一仓库的不同子目录（如 `/frontend` 与 `/backend`）分别管理，以便于部署和启动
- 代码仓库需在开题后创建
