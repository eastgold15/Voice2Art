# 任务划分 & PR 计划

> 项目：AI 语音绘图工具（题目二）
> 团队：两人（队员 A + 队员 B）

---

## 一、角色分工

| 角色 | 核心职责 | 覆盖 PR | 技术栈 |
|------|----------|---------|--------|
| **队员 A：语音交互 + LLM 集成** | Web Speech API、指令路由（正则 + LLM）、LLM API 调用、JSON 指令构造、错误降级、指令历史记录 | PR1, PR3, PR5, PR7 | Web Speech API、OpenAI API、Zustand |
| **队员 B：绘图引擎 + UI 界面** | react-konva 画布、JSON 指令执行器、shadcn/ui 工具栏、主题切换、撤销/重做、导出 PNG | PR2, PR4, PR6, PR4.5 | react-konva、shadcn/ui、Tailwind、Zustand |

### 协作约定：JSON Schema

两人通过 TypeScript Interface 约定指令格式，一人定义、另一人引用：

```typescript
// types/drawing.ts — 共同维护
export interface DrawInstruction {
  type: 'rect' | 'circle' | 'line';
  color?: string;
  strokeWidth?: number;
  size?: 'small' | 'medium' | 'large';
  position?: { x: number; y: number } | 'center' | 'leftOfPrevious' | 'rightOfPrevious';
}
```

- **A** 确保 LLM 输出符合此 Schema
- **B** 确保 `executeInstructions()` 能解析此 Schema

遵循比赛规则：每个 PR 只做一件事，粒度尽可能细，保持主分支可运行。

| PR | 内容 | 负责人 | 验收标准 |

## 三、风险与备选

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| LLM API 调用超时 | PR5-PR7 受阻 | 纯正则降级可用，不影响 PR1-PR4.5 功能 |
| API Key 耗尽 | PR5-PR7 不可用 | 切换更低价位模型（如 GPT-4o-mini 成本极低），或改用本地小模型 |
| Web Speech API 浏览器兼容 | PR1 演示受限 | 确认使用 Chrome/Edge 演示；火狐备选 |

**核心原则**：正则路径始终可用，LLM 是增值层。即使 LLM 部分出问题，基本绘图功能依然完整。
