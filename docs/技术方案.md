# 技术方案 — AI 语音绘图工具

## 一、项目概述

纯语音控制的绘图工具，用户仅通过麦克风输入自然语言指令完成绘图创作。系统采用**正则 + LLM 混合解析**架构：简单指令（改颜色、清空等）走正则快速路径，复杂自然语言指令（"画一个红色大圆，然后在它右边画一个蓝色小方块"）由 LLM 解析为结构化 JSON 后执行。

---

## 二、技术栈

| 层次 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js 15 (App Router) + TypeScript | React 生态成熟，组件化拆分支持细粒度 PR；TypeScript 强类型提升代码质量 |
| 语音识别 | Web Speech API (`SpeechRecognition`) | 纯前端、零成本、低延迟，仅需麦克风权限 |
| 绘图引擎 | `react-konva` + `konva` | React 风格封装的 Canvas 库，图形/样式/图层管理完善 |
| 状态管理 | Zustand | 轻量、无 boilerplate，天然适合"指令解析 → 更新画布"的串行逻辑 |
| UI 组件库 | shadcn/ui (Radix + Tailwind) | 管理端风格组件，内置暗色主题，观感专业 |
| 样式 | Tailwind CSS | 快速布局，与 shadcn/ui 开箱即用 |
| 代码规范 | Ultracite (Biome) | 自动格式化，保证两人协作代码风格一致，提升代码质量评分 |
| LLM API | GPT-4o-mini（降级可选 Claude 3.5 Sonnet） | 成本极低（~$0.15/百万 token），演示几天几元成本 |

## 三、系统架构

```
用户语音
    │
    ▼
┌─────────────────┐
│  Web Speech API  │  浏览器原生 ASR，实时返回识别文本
│  (SpeechRecog.)  │
└────────┬────────┘
         │ 识别文本（如"画一个红色大圆，在右边加个蓝色方块"）
         ▼
┌─────────────────┐
│  指令路由器      │  先尝试正则快速匹配，匹配失败才走 LLM
│  (Router)        │
└────┬──────┬──────┘
     │      │
     ▼      ▼
┌────────┐ ┌──────────────────────┐
│ 正则   │ │ LLM 解析器             │
│ 匹配器  │ │ (Vercel Serverless)   │
│ (毫秒) │ │ → 结构化 JSON 指令     │
└────┬───┘ └──────────┬───────────┘
     │                │
     └────────┬───────┘
              ▼ JSON 指令数组
        ┌────────────────┐
        │  Zustand Store  │  executeInstructions()
        │  (useDrawing)   │  → 更新 shapes 数组
        └────────┬───────┘
                 ▼
        ┌────────────────┐
        │  react-konva   │  自动重绘
        │  画布组件       │
        └────────────────┘
```

### 混合解析策略

| 指令类型 | 示例 | 走哪条路 |
|----------|------|----------|
| 基础图形 | "画直线"、"画圆"、"画矩形" | 正则（毫秒级） |
| 样式指令 | "红色"、"粗细5"、"清空" | 正则（毫秒级） |
| 组合指令 | "画一个红色大圆，右边画蓝色方块" | LLM（1-2 秒） |
| 模糊指令 | "画个圈"、"弄个方块" | LLM（利用语义理解） |
| 无法识别 | 无匹配项 | Toast 提示重说 |

## 四、状态管理设计（Zustand）

```typescript
// store/useDrawingStore.ts

interface Shape {
  type: 'rect' | 'circle' | 'line';
  x: number;
  y: number;
  width?: number;
  height?: number;
  radius?: number;
  points?: number[];
  fill?: string;
  stroke: string;
  strokeWidth: number;
}

interface DrawingStore {
  // 当前配置
  currentColor: string;
  currentStrokeWidth: number;

  // 图形列表（所有已绘制的图形）
  shapes: Shape[];

  // 历史记录（支持撤销/重做）
  history: Shape[][];
  historyIndex: number;

  // 指令日志（右侧面板展示）
  commandLog: string[];

  // UI 状态
  isListening: boolean;
  theme: 'light' | 'dark';

  // Actions
  setColor: (color: string) => void;
  setStrokeWidth: (width: number) => void;
  addShape: (shape: Shape) => void;
  executeInstructions: (instructions: DrawInstruction[]) => void;
  clearCanvas: () => void;
  undo: () => void;
  redo: () => void;
  recordCommand: (cmd: string) => void;
  toggleTheme: () => void;
}
```

### JSON Schema 约定（A/B 协作接口）

```typescript
// 两人共同约定的指令格式
interface DrawInstruction {
  type: 'rect' | 'circle' | 'line';
  color?: string;        // hex 颜色值，如 "#FF0000"
  strokeWidth?: number;  // 1-20
  size?: 'small' | 'medium' | 'large';
  position?: { x: number; y: number } | 'center' | 'leftOfPrevious' | 'rightOfPrevious';
}
```

## 六、创新点

1. **指令组合与顺序执行**：一次说出多步操作，系统按顺序依次执行
2. **语义容错**："画个圈" → 画圆，"画个方" → 画矩形，无法识别时 Toast 提示
3. **撤销/重做**：基于 Zustand 的 history 快照实现
4. **指令历史点击回放**：右侧面板展示所有语音指令，点击任意一条重新执行
5. **导出图片**：利用 Konva 的 `toDataURL()` 一键导出 PNG
6. **主题切换**：shadcn/ui 内置亮色/暗色主题，一键切换

## 八、部署与演示

- **本地开发**：`npm run dev` → `http://localhost:3000`（Chrome 麦克风权限）
- **部署**：Vercel 免费部署，自动 HTTPS（Web Speech API 必需）
- **运行环境**：仅需浏览器 + 麦克风，无需其他硬件
- **LLM API Key**：通过 Vercel 环境变量 `OPENAI_API_KEY` 配置，服务端运行不暴露前端
