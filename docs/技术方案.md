# 技术方案 — AI 语音绘图工具

## 一、项目概述

纯语音控制的绘图工具，用户仅通过麦克风输入自然语言指令完成绘图创作。系统采用**正则 + LLM 混合解析**架构：简单指令（改颜色、清空等）走正则快速路径，复杂自然语言指令（"画一个红色大圆，然后在它右边画一个蓝色小方块"）由 LLM 解析为结构化 JSON 后执行。

---

## 二、技术栈

| 层次 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js 15 (App Router) + TypeScript | React 生态成熟，组件化拆分支持细粒度 PR；TypeScript 强类型提升代码质量 |
| 语音识别 | Web Speech API (`SpeechRecognition`) | 纯前端、零成本、低延迟，仅需麦克风权限 |
| 绘图引擎 | `react-konva` + `konva` | React 风格封装的 Canvas 库，图形/样式/图层管理完善 |
| 状态管理 | Zustand | 轻量、无 boilerplate，天然适合"指令解析 → 更新画布"的串行逻辑 |
| UI 组件库 | shadcn/ui (Radix + Tailwind) | 管理端风格组件，内置暗色主题，观感专业 |
| 样式 | Tailwind CSS | 快速布局，与 shadcn/ui 开箱即用 |
| 代码规范 | Ultracite (Biome) | 自动格式化，保证两人协作代码风格一致，提升代码质量评分 |
| LLM API | GPT-4o-mini（降级可选 Claude 3.5 Sonnet） | 成本极低（~$0.15/百万 token），演示几天几元成本 |

## 三、系统架构

```
用户语音
    │
    ▼
┌─────────────────┐
│  Web Speech API  │  浏览器原生麦克风录音，服务端转发到阿里云 NLS  
│  (SpeechRecog.)  │
└────────┬────────┘
         │ 识别文本（如"画一个红色大圆，在右边加个蓝色方块"）
         ▼
┌─────────────────┐
│  指令路由器      │  先尝试正则快速匹配，匹配失败才走 LLM
│  (Router)        │
└────┬──────┬──────┘
     │      │
     ▼      ▼
┌────────┐ ┌──────────────────────────────────────┐
│ 正则   │ │ LLM 解析器                             │
│ 匹配器  │ │ (DeepSeek Serverless)                 │
│ (毫秒) │ │ → 获取当前画布上下文 getCanvasContext() │
└────┬───┘ │ → SYSTEM_PROMPT + 画布状态 + 用户指令  │
     │     │ → SSE 流式返回 JSON 指令                │
     │     └──────────┬───────────────────────────┘
     └────────┬───────┘
              ▼ JSON 指令（draw / pen / set-style / clear...）
        ┌───────────────────────┐
        │  executeInstructions() │  → 缓冲 → 逐条 600ms 延迟执行
        │  (Zustand Store)       │  → draw: 生成 Shape
        │                        │  → pen: 控制画笔落笔/移动/抬笔
        └──────────┬────────────┘
                   ▼
        ┌───────────────────────┐
        │  react-konva 画布      │  自动重绘（Line 支持 tension 平滑）
        └───────────────────────┘
```

### 混合解析策略

| 指令类型 | 示例 | 走哪条路 |
|----------|------|----------|
| 基础图形 | "画直线"、"画圆"、"画矩形" | 正则（毫秒级） |
| 样式指令 | "红色"、"粗细5"、"清空" | 正则（毫秒级） |
| 组合指令 | "画一个红色大圆，右边画蓝色方块" | LLM（1-2 秒） |
| 模糊指令 | "画个圈"、"弄个方块" | LLM（利用语义理解） |
| 无法识别 | 无匹配项 | Toast 提示重说 |

## 四、状态管理设计（Zustand）

```typescript
// store/useDrawingStore.ts

interface Shape {
  type: 'rect' | 'circle' | 'ellipse' | 'line';
  x: number;
  y: number;
  width?: number;
  height?: number;
  radius?: number;
  radiusX?: number;
  radiusY?: number;
  points?: number[];
  closed?: boolean;
  fill?: string;
  stroke: string;
  strokeWidth: number;
  tension?: number;        // Konva Line 平滑参数（画笔路径用）
}

interface DrawingStore {
  // 当前配置
  currentColor: string;
  currentStrokeWidth: number;

  // 图形列表 + 历史
  shapes: Shape[];
  history: Shape[][];
  historyIndex: number;

  // 指令日志
  commands: Command[];

  // 相对定位（跨 executeInstructions 调用持久化）
  lastPosition: Position | null;

  // 画笔模式状态
  penColor: string;
  penPosition: Position;   // 抽象坐标 0-1000
  penWidth: number;

  // UI 状态
  isListening: boolean;
  showGrid: boolean;
  theme: 'light' | 'dark';

  // Actions
  setColor: (color: string) => void;
  addShape: (shape: Shape) => void;
  executeInstructions: (instructions: Instruction[]) => void;
  clearCanvas: () => void;
  undo: () => void;
  redo: () => void;
  toggleTheme: () => void;
  toggleGrid: () => void;
}
```

### JSON Schema 约定（5 种指令类型）

```typescript
type Instruction = DrawShape | SetStyle | CanvasControl | PenInstruction;

interface DrawShape {          // 画规则图形
  action: "draw";
  shape: "circle" | "rectangle" | "ellipse" | "triangle" | "line";
  at: { x: number; y: number };  // 中心点坐标 0-1000
  size?: "small" | "medium" | "large";  // 默认 medium
  color?: string;              // hex 颜色
  fill?: string;               // 填充色
  strokeWidth?: number;
}

interface SetStyle {            // 设置默认样式
  action: "set-style";
  style: "color" | "stroke-width" | "fill";
  value: string | number;
}

interface CanvasControl {       // 画布控制
  action: "clear" | "undo" | "redo" | "toggle-grid";
}

interface PenInstruction {      // 画笔模式
  action: "pen";
  cmd: "move" | "down" | "up" | "color" | "width";
  at?: { x: number; y: number };  // move 目标位置
  color?: string;
  value?: number;
}
```

## 六、创新点

1. **指令组合与顺序执行**：一次说出多步操作，系统按顺序依次执行（600ms 逐条延迟动画）
2. **语义容错**："画个圈" → 画圆，"画个方" → 画矩形，无法识别时 Toast 提示
3. **撤销/重做**：基于 Zustand 的 history 快照实现
4. **指令历史点击回放**：右侧面板展示所有语音指令，点击 Play 按钮撤销到执行前并逐条重播
5. **画布上下文传给 LLM**：每次调用 LLM 都附带当前画布状态（已有图形位置、尺寸），AI 自己计算不重叠的位置
6. **画笔模式**：新增 pen 指令系统（落笔/移动/抬笔），支持画曲线、手绘、写字等不规则图形，与形状模式可混合使用
7. **导出图片**：利用 Konva 的 `toDataURL()` 一键导出 PNG
8. **主题切换**：shadcn/ui 内置亮色/暗色主题，一键切换
