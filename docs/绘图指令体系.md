# 绘图指令体系 — 核心设计

> 本文档定义 Voice2Art 的核心 JSON 指令协议，是 LLM 集成 (PR5) 和 指令执行器 (PR6) 的共同契约。
> 双方开发者请以此文档为准对齐接口。

---

## 一、整体架构

```
用户语音
   ↓
[PR1] Web Speech API → 识别文本
   ↓
[PR5] LLM（A 负责） → 将自然语言转为 JSON 指令数组
   ↓
[PR6] executeInstructions（B 负责） → 逐条解析并绘制到画布
   ↓
画布渲染
```

LLM 的角色是 **"编程者"**，我们定义的 JSON 指令就是 **"语言"**。  
LLM 根据用户说的话，组合原子指令来实现最终效果。

---

## 二、抽象坐标系（核心设计）

### 为什么需要抽象坐标系？

LLM 不知道画布的实际像素（可能是 800×600、1200×800、或全屏适配），  
所以不能要求 LLM 输出像素级坐标。

### 0-1000 抽象空间

| 概念 | 说明 |
|------|------|
| **抽象空间** | 将画布视为 `1000 × 1000` 的抽象平面 |
| **实际映射** | 执行器根据当前画布尺寸等比缩放：`实际像素 = 抽象值 ÷ 1000 × 画布尺寸` |
| **LLM 视角** | 只关心 0-1000 的抽象值，不知道像素 |

示例：

```
LLM 输出 { x: 500, y: 500 }   →   画布 1200×800 时映射到 (600px, 400px)
                                  画布 800×600  时映射到 (400px, 300px)
```

### 九宫格命名

LLM 推理"左上角""正中间"比计算坐标更自然，所以预定义 9 个锚点：

```
(0,0)         (500,0)         (1000,0)
  ┌──────────────┬──────────────┐
  │  top-left    │  top-center  │  top-right     │
  ├──────────────┼──────────────┤
  │ center-left  │   center     │  center-right  │
  ├──────────────┼──────────────┤
  │ bottom-left  │ bottom-center│ bottom-right   │
  └──────────────┴──────────────┘
(0,1000)      (500,1000)     (1000,1000)
```

锚点坐标映射：
```
top-left:      { x: 100, y: 100 }
top-center:    { x: 500, y: 100 }
center:        { x: 500, y: 500 }
bottom-right:  { x: 900, y: 900 }
... 详见 src/types/drawing.ts ANCHOR_MAP
```

### 相对定位

在上一个图形的基础上偏移：
```json
{ "relativeTo": "last", "dx": 120, "dy": 0 }
```
`dx`/`dy` 也是抽象坐标值（0-1000 范围，可正可负）。

## 三、指令类型 — 完整参考

### 3.1 画形状 — `action: "draw"`

画一个图形到画布上。支持的形状：

| ShapeKind | 说明 | 关键属性 |
|-----------|------|---------|
| `rectangle` | 矩形 | `at` 位置，`width`/`height` 或 `size` |
| `circle` | 圆形 | `at` 圆心，`radius` 或 `size` |
| `ellipse` | 椭圆 | `at` 中心，`width`/`height` 或 `size` |
| `line` | 线段 | `at` 起点 + `to` 终点 |
| `triangle` | 三角形 | `at` 位置，`size` 控制边长 |

完整字段：

```typescript
interface DrawShape {
  action: "draw";
  shape: "rectangle" | "circle" | "line" | "ellipse" | "triangle";
  at?: Location;          // 起点位置，不传 → 居中
  to?: Location;          // 终点位置（仅 line 使用）
  size?: "small" | "medium" | "large";  // 不传 → medium
  width?: number;         // 精确宽（覆盖 size）
  height?: number;        // 精确高（覆盖 size）
  radius?: number;        // circle 专用精确半径
  color?: string;         // 描边色，不传 → 用当前颜色
  fill?: string;          // 填充色，不传 → 同 color
  strokeWidth?: number;   // 不传 → 用当前粗细
}
```

#### 示例索引

```jsonc
// 最简 — 只在居中画一个默认方块
{ "action": "draw", "shape": "rectangle" }

// 指定位置 + 颜色
{ "action": "draw", "shape": "circle", "at": "top-right", "color": "#EF4444", "fill": "#EF4444", "size": "small" }

// 精确坐标 + 精确尺寸
{ "action": "draw", "shape": "rectangle", "at": { "x": 200, "y": 300 }, "width": 150, "height": 80, "color": "blue" }

// 线段 — 从左上到右下
{ "action": "draw", "shape": "line", "at": "top-left", "to": "bottom-right", "color": "black", "strokeWidth": 5 }

// 相对定位 — 在上一个旁边
{ "action": "draw", "shape": "circle", "at": "center", "size": "small", "color": "#3B82F6" }
{ "action": "draw", "shape": "circle", "at": { "relativeTo": "last", "dx": 150, "dy": 0 }, "size": "small", "color": "#22C55E" }

// 线段 + 相对定位
{ "action": "draw", "shape": "line", "at": "center-left", "to": "center-right", "color": "#7C5CFC", "strokeWidth": 4 }
```

### 3.2 改样式 — `action: "set-style"`

改变当前绘图上下文的默认值，影响后续 `draw` 指令。

```typescript
interface SetStyle {
  action: "set-style";
  style: "color" | "stroke-width" | "fill";
  value: string | number;
}
```

示例：
```json
{ "action": "set-style", "style": "color", "value": "#3B82F6" }
{ "action": "set-style", "style": "stroke-width", "value": 8 }
```

### 3.3 画布控制 — `action: "clear" | "undo" | "redo" | "toggle-grid"`

| Action | 效果 |
|--------|------|
| `clear` | 清空画布所有图形 |
| `undo` | 撤销上一步操作 |
| `redo` | 重做已撤销的操作 |
| `toggle-grid` | 显示/隐藏坐标参考网格 |

示例：
```json
{ "action": "clear" }
{ "action": "undo" }
{ "action": "toggle-grid" }
```

## 四、坐标参考网格

### 功能说明

用户需要知道抽象坐标系的位置才能准确报坐标。坐标参考网格是可切换的视觉辅助：

- **显示**：在画布上叠加半透明网格线 + 坐标标注（100 单位间隔）
- **隐藏**：恢复干净画布视图
- **切换方式**：工具栏按钮 或 语音指令 `"显示坐标"` / `"隐藏坐标"`

### 网格视觉

```
 0──100──200──300──400──500──600──700──800──900──1000
 │    │    │    │    │    │    │    │    │    │    │
 ├────┼────┼────┼────┼────┼────┼────┼────┼────┼────┤
 │    │    │    │    │    │    │    │    │    │    │  100
 ├────┼────┼────┼────┼────┼────┼────┼────┼────┼────┤
 │    │    │    │    │    │    │    │    │    │    │  200
 ...（etc，每 100 一格，半透明）
```

## 五、尺寸预设映射

| 预设 | 抽象值（0-1000 中占多少） | 说明 |
|------|--------------------------|------|
| `small` | 100 | 小元素（点、小标记） |
| `medium` | 200 | 默认大小 |
| `large` | 400 | 大元素（占据画面显著位置） |

对于 `circle`，`size` 映射到 `radius`（例如 `small` = radius 50）。
对于 `rectangle`，`size` 映射到 `width` 和 `height`（等宽等高，正方形）。
如需不等比矩形，用 `width`/`height` 精确指定。

## 七、给 A（LLM 集成）的说明

### 你的职责

在 PR5 中，你将用户语音文本送入 LLM，让 LLM 输出本协议定义的 JSON 指令数组。

### LLM Prompt 要点建议

1. **告诉 LLM 坐标范围是 0-1000**，画布被抽象化
2. **告诉 LLM 九宫格锚点**的位置（每个锚点的 xy 值）
3. **鼓励 LLM 用多条指令组合**实现复杂效果
4. **要求 LLM 用中文注释**在 JSON 里（如果不支持注释，则省略）
5. **默认值规则**：`size` 不传用 `medium`，`color` 不传用当前颜色，`at` 不传用 `center`

### 典型 LLM Prompt 片段

```
你是一个绘图指令生成器。用户用中文描述想画什么，你输出 JSON 指令数组。

坐标系统：画布抽象为 0-1000 的空间，支持九宫格位置名、
精确坐标 {x,y}、和相对定位 {relativeTo:"last", dx, dy}。

可用的形状：rectangle, circle, ellipse, line, triangle
可用的操作：draw, set-style, clear, undo, redo, toggle-grid

尺寸预设：small=100, medium=200, large=400
颜色：支持 hex (#FF0000) 和颜色名 (red)

示例：用户说"画一个红色的圆在左上角"
输出：[{"action":"draw","shape":"circle","at":"top-left","color":"#EF4444","fill":"#EF4444","size":"small"}]

请生成 JSON 指令数组，只输出 JSON，不要额外的解释。
```

### JSON 输出的预期格式

LLM 的输出应该是一个可以直接 `JSON.parse()` 的字符串：

```json
[
  { "action": "draw", "shape": "rectangle", "at": "center", "size": "large", "color": "red", "fill": "red" }
]
```

如果 LLM 输出包含 markdown 代码块，B 的 executor 需要处理（去掉 ```json 包裹）。

## 八、给 B（指令执行器）的实现指引

### executeInstructions 流程

```
输入: Instruction[]
  ↓
逐条遍历:
  ├── draw          → parseLocation(at) → 解析位置
  │                    parseLocation(to) → 解析终点（line 专用）
  │                    resolveSize()     → 尺寸映射
  │                    createShape()     → 创建 Konva Shape 对象
  │                    addShape()        → 存入 Store
  │                    addCommand()      → 记录指令历史
  │
  ├── set-style     → 更新 currentColor / currentStrokeWidth
  │
  ├── clear         → clearCanvas()
  ├── undo          → undo()
  ├── redo          → redo()
  └── toggle-grid   → 切换 showGrid 状态
```

### 坐标映射公式

```typescript
function abstractToPixel(abstractValue: number, canvasPixelSize: number): number {
  return Math.round((abstractValue / 1000) * canvasPixelSize);
}
```

### location 解析逻辑

```typescript
function parseLocation(loc: Location | undefined, lastPos: Position | null): Position {
  if (!loc) return ANCHOR_MAP.center;

  if (typeof loc === "string") {
    return ANCHOR_MAP[loc]; // 九宫格查找
  }

  if ("x" in loc && "y" in loc) {
    return loc; // 精确坐标
  }

  if ("relativeTo" in loc && lastPos) {
    return {
      x: lastPos.x + (loc.dx ?? 0),
      y: lastPos.y + (loc.dy ?? 0),
    };
  }

  return ANCHOR_MAP.center;
}
```
